# 案件選定ステータス

この文書は、パブリックコメント案件をサイトで扱う際の運用状態を定義します。
案件の優劣や回答内容の質を表すものではありません。各状態は、公式情報の確認状況と
サイトでの掲載状況を記録するために使用します。

## ステータス

| ステータス | 定義 | 公開時に記録する事実 |
|---|---|---|
| 掲載中 | 現在、サイトで案内している案件 | 公式URL、確認日、募集締切、提出先、掲載開始日 |
| 次回候補 | 対象範囲と確認条件を満たすが、現在は掲載していない案件 | 公式URL、確認日、募集締切、未掲載の運用上の理由 |
| 保留 | 募集状況、原文資料、または提出先のいずれかを確認中の案件 | 確認できている公式URL、確認日、確認待ちの項目 |
| 対象外 | 現在のフェーズの対象範囲外、または募集終了の案件 | 公式URL、確認日、対象外となった事実上の理由 |
| 取り下げ | 公式情報の更新・消失などにより、掲載を継続できなくなった案件 | 公式URL、確認日、取り下げ日、掲載を止めた事実上の理由 |

## 運用ルール

- 「落選」「重要性が低い」など、案件自体の価値判断を示す表現は使用しません。
- ステータスの変更時は、変更日、変更前後のステータス、変更理由、確認した公式情報を記録します。
- 理由には、確認できる事実のみを記載します。意見内容、行政回答、業界、政党に対する評価は記録しません。
- 「掲載中」以外の案件も台帳に残し、恣意的な削除を行いません。ただし、個人情報や公開に適さない情報は記録しません。
- 掲載案件の選定基準と同時掲載数の上限は、このステータスとは別に定めます。

## 候補の検知

- e-Govの[意見募集案件一覧RSS](https://public-comment.e-gov.go.jp/rss/pcm_list.xml)を、募集中案件の新着検知に使用します。
- RSSは候補抽出の入口であり、掲載可否を決める根拠にはしません。
- 候補を「掲載中」または「次回候補」にする前に、案件詳細、意見募集要領、案文を公式ページで確認します。
- RSSの取得日時と案件詳細の確認日時は、候補台帳に別々に記録します。

## 掲載中案件の運用

- 同時に「掲載中」にできる案件は原則1件です。
- 「掲載中」にするには、現在のPhaseの対象範囲に該当し、公式の案件詳細、意見募集要領、案文、
  締切、提出先を確認済みであることが必要です。
- 掲載は確認が完了した時点で開始します。公式締切の72時間前に新規の提出案内を終了し、
  以後は公式情報の最終確認を求めます。
- 案件ごとの要約・解説は、公式資料に記載された事実を平易に説明するものです。賛否、
  推奨、回答内容の評価は記載しません。
