# GOVERNANCE.md — 運営体制と中立性の開示（骨子）

オープンソース化が保証するのは「アルゴリズムの中立性」のみである（CLAUDE.md §3.4）。
本ドキュメントは、コード公開では担保できない以下の3点を開示するためのものである。
現時点は骨子であり、公開（M3）までに内容を確定させる。

## 1. データ取得の正確性

- 取得した行政データの出典・取得日時・ハッシュは [`SOURCES.md`](./SOURCES.md) に記録する。
- 原文は `data/raw/` に改変せず保存し、抽出・構造化されたデータとの突合レポートを公開する。
- 元資料が「主な御意見」等の抜粋である場合、その旨をデータおよび表示画面に明記する。

## 2. 運営体制

- 現在: 個人による開発・運営（発足: 2026年）。
- 意思決定: 分類ロジック・指標の変更は、変更前後の分類結果差分を公開した上で行う。
- 修正提案: GitHub Issue / Pull Request で受け付ける。マージ判断の最終権限は現状運営者にある
  （この権限の集中自体が中立性の限界であることを開示する）。
- 将来: 利用規模に応じて合議制（非営利法人型ガバナンス等）への移行を検討する。

## 3. 資金源

- 現在: 運営者の自己資金のみ。外部からの資金提供は受けていない。
- 方針: 特定の業界・政党からの資金提供は受けない（CLAUDE.md §0 Non-Goals)。
- 変更が生じた場合は本ドキュメントで開示する。

## 4. 中立性の限界（既知のもの）

完全な中立は存在しないという前提に立ち、運営の判断が入る箇所を列挙する。

| 箇所 | 判断の内容 | 緩和策 |
|---|---|---|
| 定型句リスト | どの文言を「定型句」とするか | リストと追加基準をコードで公開 |
| 類似判定しきい値 | どこから「使い回し」とするか | しきい値を公開し、変更時は差分を提示 |
| 分類カテゴリ設計 | カテゴリの粒度・「その他」の扱い | 複数案の比較検討過程を公開 |
| 対象案件の選定 | どの案件を可視化するか | 選定基準（当事者性・人数規模）を公開 |
