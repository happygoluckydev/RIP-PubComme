# RIP-PubComme

> 岩盤規制によるQoL低下を可視化し、パブコメを通じた世論形成を行うプラットフォーム

---

## 概要 / 解決したい問題

このプロジェクトが向き合う根本問題は、**岩盤規制によって行政が硬直化し、生活の質(QoL)の改善が妨げられている**ことです。

よく語られる「パブリックコメント(パブコメ)の形骸化」は、この根本問題の**症状**にすぎません。意見を出しても「ご意見として承りました」で終わり、結論が変わらない——この体験は、岩盤規制という構造を映す鏡です。

そこで本プロジェクトは、パブコメを**目的ではなく「最も効果が高い戦術」**として位置づけます。行動コストの低いパブコメ提出を入口に、生活者が「規制によってQoLが下がっている事実」を体感できる形で可視化し、世論形成につなげることを目指します。

> プロジェクトの不変の核(North Star)・根本動機・「やらないこと」の詳細は [`CLAUDE.md`](./CLAUDE.md) の **§0** を参照してください(本READMEでは重複記述しません)。

---

## スコープ(Phase)

対象は、**当事者性(困っている自覚があるか)と人数規模**を基準に選定しています。

| Phase | 対象 | 所管 |
|---|---|---|
| Phase 1 | ライドシェア・運送関連規制 | 国土交通省 |
| Phase 2 | 住宅・空き家関連の建築規制 | 国土交通省 |

以降は、同じ選定基準で対象を順次拡大していきます。

## 主な機能(MVP)

1. **不採用理由の構造的指標の可視化**
   行政の不採用理由について、内容の良し悪しは評価せず、機械的に測れる事実だけを示します。
   - 不採用理由の文字数
   - 定型句(「ご意見として承りました」等)の検出
   - 同一文言の使い回し(コピペ)検出

2. **複数指標をメニュー化したランキング**
   「不採用率」「回答までの日数」「定型句使用率」などを**別々の切り口として並列に**提示します。
   **複数指標を合成した総合スコア・総合ランキングは出しません**(単一の価値判断の押し付けを避けるため)。

3. **分類ロジックの公開(オープンソース)**
   不採用理由の分類・検出ロジックはすべて公開リポジトリ内で完結させ、ブラックボックスにしません。誰でもコードを読み、基準を確認し、改善を提案できます。

## やらないこと(Non-Goals)

中立性を守るため、以下は**意図的に行いません**。

- **総合スコア・総合ランキングを出すこと** — 「結局どこが一番悪いか」を運営が一つに決めることは、単一の価値判断の表明になるため行いません。
- **AIや運営による回答の「質」の評価・採点** — 「具体性が低い」「論理的でない」といった質的スコアリングや、「この回答は理不尽だ」といった評価ラベルの付与はしません。何が良い回答かを運営が定義した瞬間に中立性が損なわれるためです。
- **投票率向上を直接のKPIにすること** — 投票率や政治への関心は長期の社会的インパクトとして「測る」対象に留め、日々の改善は提出完了数などの行動指標で回します。

完全な中立は存在しないという前提に立ち、できる限り価値判断の混入を減らしつつ、**残る限界は開示する**という姿勢を取ります。
